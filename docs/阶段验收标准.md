## M1: 服务器核心

### 阶段 1.1 项目骨架

**目标**：可启动的 FastAPI 服务器，有完整的项目结构和配置管理。

**验收任务**：
- [ ] FastAPI 应用可启动，`/health` 端点返回 200
- [ ] 目录结构符合约定（见下方）
- [ ] 配置通过环境变量 / `.env` 文件管理
- [ ] Alembic 可执行数据库迁移（空迁移）
- [ ] `requirements.txt` / `pyproject.toml` 包含所有依赖
- [ ] Docker Compose 可启动 PostgreSQL + Redis

**验收实践**：
```bash
# 1. 启动服务器
uvicorn neuralswarm.server:app --reload
# 预期：服务器启动，无报错

# 2. 健康检查
curl http://localhost:8000/health
# 预期：{"status": "ok"}

# 3. 数据库迁移
alembic upgrade head
# 预期：迁移成功，无报错

# 4. Docker Compose
docker-compose up -d
# 预期：PostgreSQL + Redis 启动
```

**目录结构**：


### 阶段 1.2 数据库模型

**目标**：SQLAlchemy 模型定义完成，可执行迁移。

**验收任务**：
- [ ] 定义以下模型：Project, Agent, Task, Lock, Memory, AuditLog
- [ ] Alembic 生成迁移脚本并成功执行
- [ ] 模型关系正确定义（外键、一对多、多对多）
- [ ] 提供 seed data 脚本

**验收实践**：
```bash
# 1. 生成迁移
alembic revision --autogenerate -m "initial models"
# 预期：生成迁移文件

# 2. 执行迁移
alembic upgrade head
# 预期：表创建成功

# 3. 验证表结构
python -c "from neuralswarm.models import Project; print(Project.__table__.columns.keys())"
# 预期：输出列名列表

# 4. Seed data
python -m neuralswarm.seeds
# 预期：测试数据插入成功
```

**模型定义**：
```python
# 核心模型（示意）
class Project(Base):
    id, name, path, config, created_at, updated_at

class Agent(Base):
    id, project_id, name, status, capabilities, llm_config, created_at

class Task(Base):
    id, project_id, agent_id, status, input, output, created_at, completed_at

class Lock(Base):
    id, resource_uri, holder_agent_id, mode, acquired_at, expires_at

class Memory(Base):
    id, project_id, agent_id, level(L0/L1/L2/L3), content, embedding, created_at

class AuditLog(Base):
    id, agent_id, action, resource, detail, created_at
```

### 阶段 1.3 LLM 网关适配器

**目标**：可通过模型网关调用 LLM，支持 OpenAI 和 Claude 格式。

**验收任务**：
- [ ] 实现 `LLMGateway` 类，支持 OpenAI 和 Claude 格式
- [ ] 异步调用（httpx）
- [ ] 错误处理（429 限流、超时、网络错误）
- [ ] 配置文件定义模型列表和网关地址
- [ ] 单元测试覆盖

**验收实践**：
```python
# 1. 调用测试
from neuralswarm.services.llm_gateway import LLMGateway

gateway = LLMGateway(config)
response = await gateway.chat(
    model="claude-sonnet-4-6",
    messages=[{"role": "user", "content": "Hello"}]
)
# 预期：返回 LLM 响应

# 2. 限流测试
# 模拟 429 响应，验证自动降级逻辑

# 3. 格式适配测试
# 分别测试 OpenAI 格式和 Claude 格式的请求构造
```

**配置格式**：
```yaml
# config/llm_models.yaml
models:
  - id: claude-sonnet-4-6
    provider: claude
    gateway_route: claude-sonnet
    capabilities: { codegen: 4, reasoning: 4 }
    rate_limit: { rpm: 60, tpm: 100000 }

gateway:
  base_url: http://localhost:3000  # ccswitch/OneAPI 地址
  timeout: 30
```

### 阶段 1.4 Agent 运行时

**目标**：单个 Agent 可接收任务、调用 LLM、返回结果。

**验收任务**：
- [ ] `Agent` 类实现，持有上下文、工具集、LLM 配置
- [ ] Agent 可接收任务描述，调用 LLM 执行，返回结果
- [ ] Agent 状态管理（idle → running → completed / failed）
- [ ] 上下文窗口管理（截断、摘要）
- [ ] 单元测试覆盖

**验收实践**：
```python
# 1. 创建并执行 Agent
from neuralswarm.core.agent import Agent

agent = Agent(
    id="test-agent-01",
    project_id="proj-test",
    tools=["file_read", "shell"],
    llm_config={"model": "claude-sonnet-4-6"}
)
result = await agent.execute(task="列出当前目录的文件")
# 预期：Agent 调用 LLM，LLM 决定使用 shell 工具，返回文件列表

# 2. 状态检查
print(agent.status)
# 预期：completed

# 3. 上下文管理
# 发送超长任务，验证上下文截断/摘要逻辑
```

### 阶段 2.2 多 Agent 并发

**验收任务**：
- [ ] Agent 池管理（创建、销毁、回收）
- [ ] 多 Agent 并发执行不同任务
- [ ] Agent 生命周期管理（idle → running → completed → recycled）
- [ ] 并发数控制（max_concurrent_tasks）

**验收实践**：
```bash
# 同时提交 5 个任务
for i in {1..5}; do
  neuralswarm task submit "任务 $i" &
done
wait
# 预期：5 个 Agent 并发执行，无死锁
```

### 阶段 2.3 分布式锁

**验收任务**：
- [ ] Redlock 实现
- [ ] X 锁（独占）、S 锁（共享读）、IX/IS 锁（意向）
- [ ] URI 资源标识（`project://xxx/src/utils.py`）
- [ ] 锁超时和自动释放
- [ ] 锁信息持久化到 PostgreSQL

**验收实践**：
```python
# 1. 获取锁
lock = await lock_manager.acquire("project://test/src/main.py", mode="X", agent_id="agent-01")
# 预期：获取成功

# 2. 冲突测试
lock2 = await lock_manager.acquire("project://test/src/main.py", mode="X", agent_id="agent-02")
# 预期：获取失败，返回冲突信息

# 3. 释放锁
await lock_manager.release(lock)
# 预期：释放成功
```

### 阶段 2.4 锁冲突通知

**验收任务**：
- [ ] WebSocket 端点 `/ws/conflicts`
- [ ] 锁冲突时推送 `scheduling_request` 事件
- [ ] 用户可通过 WebSocket 发送决策（抢占/排队/合并/手动）
- [ ] 超时自动执行默认策略

**验收实践**：
```bash

wscat -c ws://localhost:8000/ws/conflicts

# 2. 触发锁冲突（两个 Agent 同时操作同一文件）
# 预期：WebSocket 收到 scheduling_request 事件

# 3. 发送决策
# 预期：决策被执行，锁状态更新
```
