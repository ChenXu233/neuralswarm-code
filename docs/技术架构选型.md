# NeuralSwarm/Code 技术架构选型

## 总体分工

| 层 | 技术 | 职责 |
|---|------|------|
| **服务器** | Python / FastAPI | 调度、Agent 运行时、模型网关对接、MCP 桥接、锁、记忆、安全 |
| **客户端 Core** | Rust | 本地资源访问、SSH 反向隧道、与服务器通信、执行指令 |
| **桌面端 UI** | Tauri v2 + Vue + TypeScript | Agent 状态、锁冲突通知、Markdown 渲染、悬浮窗 |
| **移动端 UI** | Jetpack Compose (Android) / SwiftUI (iOS) | 同桌面端功能，适配移动端交互 |
| **Web UI** | Vue + TypeScript | 复用桌面端 UI 组件，浏览器访问 |

**设计原则**：
- 服务器是"脑"（调度、推理、记忆），用 Python 生态（LLM SDK 最成熟）
- 客户端是"手"（执行、暴露资源），用 Rust（系统级能力、单二进制部署）
- UI 层通过本地 API 与客户端 Core 通信，各端独立实现

## 服务器技术栈

| 组件 | 技术 | 说明 |
|------|------|------|
| Web 框架 | FastAPI | 异步高并发，原生 WebSocket 支持 |
| Agent 运行时 | asyncio | 每个 Agent 一个协程，Python LLM SDK 原生支持 |
| 模型网关对接 | httpx (async) | 适配 OpenAI/Claude 格式，兼容 ccswitch/OneAPI/NewAPI |
| MCP 桥接网关 | mcp Python SDK | 官方 MCP SDK，对接 MCP Server |
| 分布式锁 | redis-py + Redlock | Redis 官方客户端，Redlock 算法 |
| 事件总线 | NATS (nats.py) 或 Redis Streams | NATS 轻量高性能；Redis Streams 减少组件 |
| 向量数据库 | Qdrant (qdrant-client) | L2 语义记忆、Spec 向量检索 |
| 关系数据库 | PostgreSQL (SQLAlchemy + asyncpg) | 锁持久化、审计日志、L3 过程性记忆 |
| 缓存 | Redis (redis-py) | L0 瞬时记忆、L1 情景记忆、Agent 状态 |
| 选主 | etcd (etcd3-py) | 调度器高可用选主 |
| 安全 | FastAPI middleware + Python ssl | RBAC 中间件 + mTLS |

## 客户端技术栈

### Core（Rust）

| 组件 | Crate | 说明 |
|------|-------|------|
| WebSocket 客户端 | `tokio-tungstenite` | 连接服务器，接收指令 |
| SSH 反向隧道 | `thrussh` / `ssh2` | 建立到服务器的反向隧道 |
| 文件操作 | `std::fs` + `notify` | 读写文件 + 变更监听 |
| Shell 执行 | `tokio::process` | 执行命令并捕获输出 |
| 剪贴板 | `arboard` | 跨平台剪贴板访问 |
| 系统通知 | `notify-rust` | 锁冲突等事件通知 |
| HTTP 服务器 | `axum` | 本地 API，供 UI 层调用 |

### 桌面端 UI（Tauri v2 + Vue）

| 组件 | 技术 | 说明 |
|------|------|------|
| 桌面壳 | Tauri v2 | Rust 后端 + Web 前端，~5MB |
| 前端框架 | Vue 3 + TypeScript | 响应式 UI |
| Markdown 渲染 | markdown-it + shiki | 完整渲染 + 语法高亮 |
| 实时通信 | WebSocket | 与客户端 Core 本地 API 通信 |
| UI 组件 | Naive UI / Element Plus | Vue 生态组件库 |

### 移动端 UI

| 平台 | 技术 | 说明 |
|------|------|------|
| Android | Jetpack Compose + Kotlin | 原生 Android UI |
| iOS | SwiftUI | 原生 iOS UI |

移动端与客户端 Core 通过本地 HTTP/WebSocket API 通信，复用相同的通信协议。

## 通信协议

### 服务器 ↔ 客户端 Core

```json
// 服务器 → 客户端：执行指令
{
  "type": "execute",
  "command": "file_read",
  "params": { "path": "/home/user/project/main.py" },
  "request_id": "req-001"
}

// 客户端 → 服务器：执行结果
{
  "type": "result",
  "request_id": "req-001",
  "status": "success",
  "data": { "content": "..." }
}

// 服务器 → 客户端：锁冲突通知
{
  "type": "notification",
  "event": "lock_conflict",
  "data": {
    "resource": "project://xxx/src/utils.py",
    "holder": "Agent-B",
    "requester": "Agent-A",
    "actions": ["preempt", "queue", "merge", "manual"]
  }
}

// 客户端 → 服务器：用户决策
{
  "type": "decision",
  "request_id": "req-002",
  "action": "preempt"
}
```

### 客户端 Core ↔ UI

UI 通过 localhost HTTP/WebSocket 连接客户端 Core，复用上述协议格式。
