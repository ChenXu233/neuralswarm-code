# NeuralSwarm/Code 技术架构选型

## 总体分工

| 层 | 技术 | 职责 |
|---|------|------|
| **Client Core** | Rust | 本地资源访问、MCP Server 暴露、离线缓存 |
| **服务器** | Python / FastAPI | LLM 调度、Agent 运行时、MCP 客户端、记忆、安全 |
| **桌面端 UI** | Tauri v2 + Vue + TypeScript | Agent 状态、冲突通知、Markdown 渲染、悬浮窗 |
| **移动端 UI** | Jetpack Compose (Android) / SwiftUI (iOS) | 同桌面端功能，适配移动端交互 |
| **Web UI** | Vue + TypeScript | 复用桌面端 UI 组件，浏览器访问 |

**设计原则**：
- Client Core 是"手"（资源暴露、本地执行），用 Rust（系统级能力、单二进制部署）
- 服务器是"脑"（LLM 调度、Agent 运行时），用 Python 生态（LLM SDK 最成熟）
- UI 层通过本地 API 与 Client Core 通信，各端独立实现
- MCP 协议统一资源访问，客户端暴露本地能力，服务器调用这些能力

## 服务器技术栈

| 组件 | 技术 | 说明 |
|------|------|------|
| Web 框架 | FastAPI | 异步高并发，原生 WebSocket 支持 |
| Agent 运行时 | asyncio | 每个 Agent 一个协程，Python LLM SDK 原生支持 |
| 模型网关对接 | httpx (async) | 适配 OpenAI/Claude 格式，兼容 ccswitch/OneAPI/NewAPI |
| MCP 客户端 | mcp Python SDK | 官方 MCP SDK，调用 Client Core MCP Server |
| 并发控制 | Git worktree + 文件哈希 | 任务隔离 + 乐观并发控制 |
| 事件总线 | Redis Streams | 事件发布/订阅 |
| 关系数据库 | PostgreSQL (SQLAlchemy + asyncpg) | 审计日志、记忆存储 |
| 缓存 | Redis (redis-py) | Agent 状态、项目注册表 |
| 安全 | FastAPI middleware + Python ssl | RBAC 中间件 + mTLS |

## 客户端技术栈

### Core（Rust）

| 组件 | Crate | 说明 |
|------|-------|------|
| MCP Server | 自定义实现 | 暴露本地能力为 MCP 工具 |
| WebSocket 服务器 | `tokio-tungstenite` | MCP over WebSocket 传输层 |
| 文件操作 | `std::fs` + `notify` | 读写文件 + 变更监听 |
| Shell 执行 | `tokio::process` | 执行命令并捕获输出 |
| Git 操作 | `git2` | Git 库操作 |
| 剪贴板 | `arboard` | 跨平台剪贴板访问 |
| 系统通知 | `notify-rust` | 冲突等事件通知 |
| HTTP 服务器 | `axum` | 本地 API，供 UI 层调用 |

### 桌面端 UI（Tauri v2 + Vue）

| 组件 | 技术 | 说明 |
|------|------|------|
| 桌面壳 | Tauri v2 | Rust 后端 + Web 前端，~5MB |
| 前端框架 | Vue 3 + TypeScript | 响应式 UI |
| Markdown 渲染 | markdown-it + shiki | 完整渲染 + 语法高亮 |
| 实时通信 | WebSocket | 与客户端 Core 本地 API 通信 |
| UI 组件 | Naive UI / Element Plus | Vue 生态组件库 |

### 移动端 UI

| 平台 | 技术 | 说明 |
|------|------|------|
| Android | Jetpack Compose + Kotlin | 原生 Android UI |
| iOS | SwiftUI | 原生 iOS UI |

移动端与客户端 Core 通过本地 HTTP/WebSocket API 通信，复用相同的通信协议。

## 通信协议

### 服务器 ↔ Client Core（MCP 协议）

使用 MCP（Model Context Protocol） over WebSocket 进行通信：

```json
// 服务器 → Client Core：MCP 工具调用
{
  "type": "tool_call",
  "tool": "file_read",
  "params": { "path": "/home/user/project/main.py" },
  "project": "proj-xxx",
  "request_id": "req-001"
}

// Client Core → 服务器：MCP 工具结果
{
  "type": "tool_result",
  "request_id": "req-001",
  "status": "success",
  "data": { "content": "..." }
}

// 服务器 → Client Core：文件冲突通知
{
  "type": "notification",
  "event": "file_conflict",
  "data": {
    "conflict_id": "conflict-001",
    "file_path": "src/utils.py",
    "agent_id": "agent-2",
    "other_agent_id": "agent-1",
    "old_hash": "abc123",
    "current_hash": "def456",
    "current_content": "...",
    "new_content": "...",
    "actions": ["re_read", "overwrite", "submit_to_scheduler"]
  }
}

// Client Core → 服务器：用户决策
{
  "type": "decision",
  "request_id": "req-002",
  "action": "preempt"
}
```

### 客户端 Core ↔ UI

UI 通过 localhost HTTP/WebSocket 连接客户端 Core，复用上述协议格式。
