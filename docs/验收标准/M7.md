# M7: 高级功能

**目标**：全局聊天模式（无项目绑定）、Mesh 互联（跨服务器资源访问）、调度器高可用。

**可见产出**：全局聊天可跨项目派发任务，服务器间可互访资源。

---

## 阶段 7.1 全局聊天

**目标**：无项目绑定的聊天模式，可跨项目派发任务。

**验收任务**：
- [ ] 全局聊天空间（不关联任何项目）
- [ ] 独立的 L0-L3 记忆和 Spec（与项目隔离）
- [ ] 跨项目任务派发：
  - 用户在全局聊天中说 "帮 proj-a 重构数据库连接"
  - 系统在 proj-a 空间创建任务并执行
  - 结果返回全局聊天
- [ ] 只派发不阻塞（派发后立即返回，任务后台执行）
- [ ] 手动"钉入"机制（将全局对话钉入指定项目的 L1 记忆）
- [ ] 全局聊天 API：`/api/global-chat/messages`
- [ ] 单元测试覆盖

**验收实践**：
```bash
# 1. 全局聊天提交
curl -X POST http://localhost:8000/api/global-chat/messages \
  -d '{"content": "帮我 proj-a 重构数据库连接池"}'
# 预期：立即返回，任务在后台执行

# 2. 查看派发状态
curl http://localhost:8000/api/global-chat/tasks
# 预期：显示派发的任务和状态

# 3. 钉入对话
curl -X POST http://localhost:8000/api/global-chat/pin \
  -d '{"message_id": "msg-001", "project_id": "proj-a"}'
# 预期：消息钉入 proj-a 的 L1 记忆
```

---

## 阶段 7.2 Mesh 互联

**目标**：服务器可对接其他 Mesh 网络，客户端可跨 Mesh 访问资源。

**验收任务**：
- [ ] Mesh 节点注册和发现
- [ ] 跨 Mesh 资源路由：
  - 客户端 A 在 Mesh 1
  - 客户端 B 在 Mesh 2
  - Agent 可通过 Mesh 互联访问客户端 B 的资源
- [ ] 路由表管理（哪些资源在哪个 Mesh）
- [ ] 跨 Mesh 通信协议（WebSocket / HTTP）
- [ ] 故障隔离（一个 Mesh 故障不影响其他 Mesh）
- [ ] 单元测试覆盖

**验收实践**：
```bash
# 1. 启动 Mesh 1
uvicorn neuralswarm.server:app --port 8000

# 2. 启动 Mesh 2
uvicorn neuralswarm.server:app --port 8001

# 3. Mesh 1 注册 Mesh 2
curl -X POST http://localhost:8000/api/mesh/peers \
  -d '{"url": "http://localhost:8001", "name": "mesh-2"}'

# 4. 通过 Mesh 互联访问资源
# Agent in Mesh 1 访问 client://mesh-2-laptop/path/to/file
# 预期：通过 Mesh 互联，返回 Mesh 2 客户端的文件
```
