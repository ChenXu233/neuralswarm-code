# M7: 高级功能 + 移动端

**状态**：待开始

**目标**：实现全局聊天模式和移动端适配，让 AI 能够跨项目协作并在移动设备上使用。

---

## 用户故事

**作为**开发者，
**我想要**在全局聊天中跨项目派发任务，并且能在手机上使用 NeuralSwarm，
**以便**我能随时随地管理多个项目的 AI 任务，不受设备限制。

**示例**：
- "帮 proj-a 重构数据库连接" → AI 在 proj-a 空间执行任务，结果返回全局聊天
- "帮 proj-b 添加用户认证" → AI 在 proj-b 空间执行任务，结果返回全局聊天
- 在手机上查看任务状态、接收通知、快速响应冲突

---

## E2E 流程

### 流程 1：全局聊天跨项目派发

```
用户操作：
1. 打开桌面应用
2. 进入全局聊天模式
3. 输入 "帮 proj-a 重构数据库连接池"
4. 点击发送

系统响应：
5. 系统识别项目名称 "proj-a"
6. 在 proj-a 空间创建任务
7. 任务在后台异步执行
8. 立即返回确认消息
9. 任务完成后，结果返回全局聊天

用户看到：
- 立即收到确认："已派发任务到 proj-a"
- 任务状态实时更新
- 任务完成后，看到完整结果
- 可以继续派发其他任务，不阻塞
```

### 流程 2：钉入对话到项目

```
用户操作：
1. 在全局聊天中讨论 proj-a 的架构设计
2. 觉得讨论结果有价值
3. 点击 "钉入" 按钮
4. 选择目标项目 "proj-a"

系统响应：
5. 将对话内容钉入 proj-a 的 L1 记忆
6. 后续 proj-a 的 Agent 可以检索到这段对话

用户看到：
- 钉入成功提示
- 在 proj-a 的记忆面板中看到钉入的对话
- 后续 proj-a 的任务自动引用这段对话
```

### 流程 3：移动端使用

```
用户操作：
1. 在手机上打开 NeuralSwarm 应用
2. 查看任务列表
3. 收到通知："proj-a 的任务完成"
4. 点击查看结果
5. 快速响应冲突决策

系统响应：
6. 移动端通过本地 API 连接 Client Core
7. 实时同步任务状态
8. 推送通知
9. 支持快速决策

用户看到：
- 任务列表（按项目分组）
- 任务状态实时更新
- 推送通知
- 冲突决策界面
- 任务结果查看
```

## 完成定义（DoD）

### 全局聊天要求
- [ ] 全局聊天空间（不关联任何项目）
- [ ] 独立的 L0-L3 记忆和 Spec（与项目隔离）
- [ ] 跨项目任务派发：
  - 用户在全局聊天中说 "帮 proj-a 重构数据库连接"
  - 系统在 proj-a 空间创建任务并执行
  - 结果返回全局聊天
- [ ] 只派发不阻塞（派发后立即返回，任务后台执行）
- [ ] 手动"钉入"机制（将全局对话钉入指定项目的 L1 记忆）

## 技术要求

### Server (Python)

| 组件 | 说明 |
|------|------|
| 全局聊天 API | `/api/global-chat/messages` |
| 跨项目派发 | 识别项目名称，创建任务 |
| 钉入机制 | 对话钉入项目 L1 记忆 |
| 推送通知 | WebSocket 推送 + 移动端推送 |

### Client Core (Rust)

| 组件 | 说明 |
|------|------|
| 本地 API | 供移动端调用 |
| 离线缓存 | 任务列表、历史记录缓存 |
| 推送通知 | 系统通知集成 |

### UI (Vue + Tauri)

| 组件 | 说明 |
|------|------|
| 全局聊天界面 | 无项目绑定的对话 |
| 跨项目派发状态 | 任务派发进度展示 |
| 钉入操作 UI | 选择项目 → 钉入对话 |
| 调度器状态 | 服务器实例、负载 |

### 移动端 (Android / iOS)

| 组件 | 说明 |
|------|------|
| 任务列表 | 按项目分组、状态筛选 |
| 任务详情 | 结果查看、操作按钮 |
| 推送通知 | 任务完成、冲突决策 |
| 冲突决策 | 快速响应界面 |
| 离线浏览 | 基本任务查看 |
