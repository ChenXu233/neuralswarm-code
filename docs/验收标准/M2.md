# M2: MVP ✅

**状态**：已完成（92 个测试通过）

**目标**：用户可通过桌面应用打开项目文件夹、提交任务、实时看到 Agent 对话流和执行状态。

**可见产出**：Tauri 桌面应用能跑起来，打开项目 → 提交任务 → 实时看到 Agent 的工具调用和回复。

**并行轨道**：
- Track A（服务器）：Project API + Task API + Redis + WebSocket
- Track C（UI）：Vue 前端 + Tauri 壳

**关键设计决策**：
- Project 是真实文件夹，用户通过 UI "打开文件夹" 注册（类似 VSCode）
- MVP 仅支持服务器本地项目，客户端远程项目在 M3 支持
- Agent 后台异步执行（asyncio.Task），通过 WebSocket 推送进度
- `client/` 是三端共用的 Vue 前端，`desktop/` 是 Tauri 壳复用 `client/`

## 阶段 2.1 Project API

**目标**：项目注册和管理，项目对应服务器上的真实文件夹。

**验收任务**：
- [x] `POST /api/projects` — 注册项目（指定服务器本地路径）
- [x] `GET /api/projects` — 列出项目
- [x] `GET /api/projects/{id}` — 项目详情
- [x] `DELETE /api/projects/{id}` — 删除项目（不删除文件夹）
- [x] 路径验证（路径存在、是目录、有读写权限）
- [x] 项目下任务列表：`GET /api/projects/{id}/tasks`
- [x] Pydantic schema 定义
- [x] 错误处理（路径不存在、重复注册）
- [x] 单元测试覆盖

**验收实践**：
```bash
# 注册项目
curl -X POST http://localhost:8000/api/projects \
  -H "Content-Type: application/json" \
  -d '{"name": "my-project", "path": "/home/user/my-project"}'
# 预期：{"id": "proj-xxx", "name": "my-project", "path": "server:///home/user/my-project", ...}

# 列出项目
curl http://localhost:8000/api/projects
# 预期：{"items": [...], "total": N}
```

## 阶段 2.2 Task API

**目标**：任务提交、查询、取消。Agent 后台异步执行。

**验收任务**：
- [x] `POST /api/tasks` — 提交任务（需指定 project_id，prompt）
- [x] `GET /api/tasks/{id}` — 查询任务状态和结果
- [x] `GET /api/tasks` — 列出任务（支持分页、按 project_id 过滤）
- [x] `DELETE /api/tasks/{id}` — 取消任务（中断 Agent 执行）
- [x] 默认调度：自动分配 Agent 和 LLM（seed data 中的默认值）
- [x] Agent 后台异步执行（asyncio.Task）
- [x] 工具元数据：`ToolMetadata` + `ToolParameter`，支持 OpenAI/Anthropic function calling 格式转换
- [x] Agent 执行时将工具 schema 传给 LLM，解析工具调用并执行
- [x] 任务状态流转：pending → running → completed / failed / cancelled
- [x] Pydantic schema 定义（请求/响应模型，统一错误格式）
- [x] 错误处理（404、422、500）
- [x] 单元测试覆盖

**验收实践**：
```bash
# 提交任务
curl -X POST http://localhost:8000/api/tasks \
  -H "Content-Type: application/json" \
  -d '{"project_id": "proj-xxx", "prompt": "列出当前目录的文件"}'
# 预期：{"id": "task-xxx", "status": "pending", ...}

# 查询状态
curl http://localhost:8000/api/tasks/task-xxx
# 预期：{"id": "task-xxx", "status": "completed", "output": "..."}

# 取消任务
curl -X DELETE http://localhost:8000/api/tasks/task-xxx
# 预期：{"id": "task-xxx", "status": "cancelled"}
```

## 阶段 2.4 WebSocket 实时推送

**目标**：WebSocket 端点实时推送 Agent 对话流和状态变更。

**验收任务**：
- [x] WebSocket 端点 `ws://localhost:8000/ws/tasks/{task_id}`
- [x] 推送事件类型：
  - `status` — 状态变更（pending → running → completed / failed）
  - `tool_call` — 工具调用（工具名、参数）
  - `tool_result` — 工具结果（输出）
  - `message` — Agent 输出消息
  - `error` — 错误信息
- [x] 断线重连后补发错过的事件（从 Redis 读取）
- [x] 多客户端同时订阅同一任务
- [x] 心跳检测
- [x] 单元测试覆盖

**验收实践**：
```bash
# 连接 WebSocket
wscat -c ws://localhost:8000/ws/tasks/task-xxx

# 提交任务后，预期收到：
# {"type": "status", "data": {"status": "running"}}
# {"type": "tool_call", "data": {"tool": "shell", "args": "ls -la"}}
# {"type": "tool_result", "data": {"tool": "shell", "output": "file1.py\nfile2.py"}}
# {"type": "message", "data": {"content": "当前目录有以下文件..."}}
# {"type": "status", "data": {"status": "completed"}}
```
