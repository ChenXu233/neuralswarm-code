# M5: MCP 协议 + 认知层

**状态**：待开始

**目标**：实现 MCP 协议统一资源访问，同时实现多层记忆和事件总线。

---

## 用户故事

**作为**开发者，
**我想要**通过自然语言提交任务，让 AI 自动操作我的本地文件和代码，并且 AI 能记住之前的对话和项目知识，
**以便**我能专注于设计和思考，而不是重复性的编码工作，并且 AI 能够基于项目上下文给出更好的建议。

**示例**：
- "列出当前目录的文件" → AI 执行 shell 命令，返回文件列表
- "读取 src/main.py 的内容" → AI 读取文件，返回代码
- "重构 utils.py 中的函数，添加错误处理" → AI 读取代码、理解结构、执行重构
- "给项目添加用户认证" → AI 记住项目使用 FastAPI + SQLAlchemy，自动基于已有知识执行任务

## E2E 流程

### 流程 1：基本 MCP 工具调用

```
用户操作：
1. 打开桌面应用
2. 选择本地项目文件夹
3. 输入 "列出当前目录的文件"
4. 点击发送

系统响应：
5. 应用将任务发送到服务器
6. 服务器调用 LLM 分析任务
7. LLM 决定调用 shell_execute 工具
8. 服务器通过 MCP 调用 Client Core 的 shell_execute
9. Client Core 执行 "ls -la" 命令
10. Client Core 返回命令输出
11. 服务器将结果返回给 LLM
12. LLM 生成人类可读的回复
13. 应用显示 Agent 的回复

用户看到：
- 实时显示 Agent 的思考过程
- 显示工具调用（shell: ls -la）
- 显示命令输出
- 显示 Agent 的最终回复
- 任务状态变为 "completed"
```

### 流程 2：记忆增强的任务执行

```
用户操作：
1. 提交任务 A："创建一个 Python 项目结构"
2. 等待任务完成
3. 提交任务 B："给项目添加用户认证"

系统响应（任务 A）：
4. Agent 执行任务，创建项目结构
5. L1 记录事件："创建了 FastAPI + SQLAlchemy 项目结构"
6. 反思 Agent 异步整理，L2 生成知识："项目使用 FastAPI + SQLAlchemy"

系统响应（任务 B）：
7. Agent 从 L2 检索到 "项目使用 FastAPI + SQLAlchemy"
8. Agent 从 L3 检索到用户偏好："代码风格：简洁，不要多余注释"
9. 自动注入到上下文
10. Agent 基于已有知识执行任务，自动使用 FastAPI + SQLAlchemy 实现认证

用户看到：
- 任务 A 完成，项目结构创建成功
- 任务 B 完成，认证功能基于 FastAPI + SQLAlchemy 实现
- Agent 记住了项目结构，没有重复询问
```

### 流程 3：事件驱动协作

```
用户操作：
1. 提交任务 A："修改 src/utils.py 的 process_data 函数"
2. 提交任务 B："审查 src/utils.py 的代码质量"

系统响应：
3. Agent A 执行任务，修改了 src/utils.py
4. Agent A 发布事件 "file:changed" 到事件总线
5. Agent B 订阅了 "file:changed" 事件
6. Agent B 收到事件，自动触发代码审查
7. Agent B 生成审查报告

用户看到：
- 任务 A 完成，代码被修改
- 任务 B 自动触发，生成审查报告
- 两个任务并行执行，互不阻塞
```

## 完成定义（DoD）

### MCP 协议要求
- [ ] Client Core 可以作为 MCP Server 启动
- [ ] Client Core 暴露以下 MCP 工具：
  - `file_read` — 读取文件内容
  - `file_write` — 写入文件
  - `shell_execute` — 执行 Shell 命令
  - `git_log` — 查看 Git 历史
  - `git_diff` — 查看 Git diff
- [ ] Server 可以作为 MCP Client 连接 Client Core
- [ ] Server 可以通过 MCP 调用 Client Core 的工具
- [ ] LLM 可以决定调用哪些 MCP 工具
- [ ] LLM 可以基于工具结果生成回复

### 认知层要求
- [ ] L0 瞬时记忆：Agent 上下文持久化，重启后可恢复
- [ ] L1 情景记忆：项目事件流，带 TTL（24 小时）
- [ ] L2 语义记忆：长期知识，由反思 Agent 异步整理
- [ ] L3 过程性记忆：用户偏好，全局作用域
- [ ] 事件总线：基于 Redis Streams，支持发布/订阅

### 用户体验要求
- [ ] 用户提交任务后，5 秒内看到 Agent 开始响应
- [ ] 工具调用过程实时显示（不阻塞 UI）
- [ ] 错误信息清晰可理解（不是技术性错误）
- [ ] 任务状态实时更新（pending → running → completed）
- [ ] Agent 能够记住之前的对话和项目知识

### 技术要求
- [ ] MCP 协议基于 WebSocket 传输
- [ ] 工具参数使用 JSON Schema 验证
- [ ] 错误处理覆盖所有异常情况
- [ ] 单元测试覆盖率 > 80%

## 技术要求

### Client Core (Rust)

| 组件 | 说明 |
|------|------|
| MCP Server | 暴露本地能力为 MCP 工具 |
| WebSocket 服务器 | MCP over WebSocket 传输层 |
| 文件操作 | 读写文件、监听变更 |
| Shell 执行 | 执行命令并捕获输出 |
| Git 操作 | 查看历史、分支、diff |
| 参数验证 | JSON Schema 验证 |
| 错误处理 | 工具不存在、参数错误、执行失败 |

### Server (Python)

| 组件 | 说明 |
|------|------|
| MCP 客户端 | 调用 Client Core MCP Server |
| WebSocket 连接管理 | 连接、断开、重连 |
| 项目路由 | project → client 映射 |
| 工具调用封装 | 统一的调用接口 |
| LLM 集成 | 将 MCP 工具注册为 Agent 工具 |
| 多层记忆 | L0-L3 记忆系统 |
| 事件总线 | Redis Streams 发布/订阅 |
| 反思 Agent | 异步整理 L0 + L1 → L2 |

### UI (Vue + Tauri)

| 组件 | 说明 |
|------|------|
| 任务提交界面 | 输入框 + 发送按钮 |
| 对话流展示 | 显示 Agent 消息和工具调用 |
| 工具调用可视化 | 实时显示工具执行过程 |
| 任务状态指示 | pending / running / completed / failed |
| 记忆面板 | 按层级展示 L0-L3 |
| 事件流时间线 | 类似 Git log |
