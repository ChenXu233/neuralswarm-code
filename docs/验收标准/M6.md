# M6: 调度 + 安全

**状态**：待开始

**目标**：实现智能任务调度和安全层，让 AI 能够自动拆解复杂任务并匹配最优 Agent+LLM 组合。

---

## 用户故事

**作为**开发者，
**我想要**提交复杂任务后，AI 能够自动拆解为子任务并分配给最合适的 Agent+LLM 组合，
**以便**我能专注于高层设计，而不是手动管理任务执行顺序和资源分配。

**示例**：
- "给项目添加用户认证功能" → AI 自动拆解为：设计架构、实现注册、实现登录、JWT 中间件、集成测试
- "重构数据库连接池" → AI 自动拆解为：分析现有代码、设计新方案、实现重构、验证测试
- "优化 API 性能" → AI 自动拆解为：性能分析、瓶颈识别、优化实现、性能测试

---

## E2E 流程

### 流程 1：复杂任务自动拆解

```
用户操作：
1. 打开桌面应用
2. 选择项目
3. 输入 "给项目添加用户认证功能，包括注册、登录、JWT 中间件"
4. 点击发送

系统响应：
5. 调度器接收任务
6. 使用轻量 LLM 分析任务，生成子任务 DAG：
   - 子任务 1：设计认证架构（无依赖）
   - 子任务 2：实现注册接口（依赖 1）
   - 子任务 3：实现登录接口（依赖 1）
   - 子任务 4：实现 JWT 中间件（依赖 1）
   - 子任务 5：集成测试（依赖 2, 3, 4）
7. 调度器匹配 Agent+LLM：
   - 子任务 1 → Agent-Review + Claude Opus（需要强推理）
   - 子任务 2-4 → Agent-Code + Claude Sonnet（代码生成）
   - 子任务 5 → Agent-Test + DeepSeek（测试，成本低）
8. 按 DAG 顺序执行子任务
9. 无依赖的子任务并行执行

用户看到：
- DAG 可视化（任务依赖图）
- 子任务状态（pending / running / completed / failed）
- Agent+LLM 匹配结果
- 执行进度实时更新
- 最终结果：认证功能完整实现
```

### 流程 2：LLM 限流自动切换

```
用户操作：
1. 提交任务 "重构数据库连接池"

系统响应：
2. 调度器分配任务给 Agent + Claude Sonnet
3. Agent 开始执行，调用 LLM
4. LLM 返回 429 限流错误
5. 调度器检测到限流，标记 Claude Sonnet 为 throttled
6. 调度器自动切换到 DeepSeek（能力等级降级）
7. Agent 使用 DeepSeek 继续执行
8. 任务完成

用户看到：
- 任务开始执行
- LLM 切换提示（Claude Sonnet → DeepSeek）
- 任务完成
- LLM 切换历史记录
```

### 流程 3：安全认证和权限控制

```
用户操作：
1. 首次启动应用
2. 系统提示输入 API Key 和服务器地址
3. 用户输入配置
4. 系统验证配置并建立安全连接

系统响应：
5. 客户端和服务器建立 mTLS 连接
6. 服务器验证客户端证书
7. 用户角色确定（admin / operator / viewer）
8. 根据角色限制 API 访问权限

用户看到：
- 安全连接建立成功
- 用户角色显示
- 功能根据权限显示/隐藏
```

## 完成定义（DoD）

### 调度器要求
- [ ] 任务解构：使用轻量 LLM 将用户任务分解为子任务 DAG
- [ ] Agent+LLM 能力匹配：
  - Agent 能力：工具集覆盖度
  - LLM 能力：推理/代码生成等级
  - 匹配评分函数：`Score(A, L, T) = agent_match(A, T) * llm_match(L, T) * g(load_A)`
- [ ] LLM 自动切换：
  - 限流检测（429 → 标记 throttled）
  - 按能力等级降级
  - Token 消耗均衡
  - 恢复探测
- [ ] DAG 执行引擎：
  - 按依赖顺序执行子任务
  - 无依赖的子任务并行执行
  - 失败处理（重试 / 降级 / 通知用户）

### 安全层要求
- [ ] mTLS 证书：
  - 自动生成自签名 CA
  - 服务器和客户端证书生成
  - 证书验证（连接时校验证书链）
- [ ] RBAC 中间件：
  - 角色定义：admin、operator、viewer
  - 权限矩阵：
    | 操作 | admin | operator | viewer |
    |------|-------|----------|--------|
    | 提交任务 | ✅ | ✅ | ❌ |
    | 查看任务 | ✅ | ✅ | ✅ |
    | 管理 Agent | ✅ | ❌ | ❌ |
    | 冲突决策 | ✅ | ✅ | ❌ |
  - API 端点权限标注
- [ ] 操作审计日志：
  - 结构化日志（JSON 格式）
  - 记录：agent_id、action、resource、detail、timestamp
  - 不可篡改（append-only）

### 用户体验要求
- [ ] 复杂任务自动拆解，用户无需手动管理子任务
- [ ] LLM 限流时自动切换，用户无感知
- [ ] 安全连接建立，用户配置简单
- [ ] 权限控制清晰，用户知道能做什么

### 技术要求
- [ ] 调度器支持 DAG 任务拆解
- [ ] Agent+LLM 能力声明式配置
- [ ] mTLS 证书自动生成和管理
- [ ] RBAC 中间件实现
- [ ] 审计日志记录
