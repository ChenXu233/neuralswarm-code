# M3: 资源访问

**目标**：Agent 可通过 MCP 桥接网关和 SSH 隧道访问客户端本地资源（文件、Shell）。支持注册本地项目。

**可见产出**：Agent 能读写客户端本地文件、执行 Shell 命令，桌面端可打开本地项目。

**关键变化**：M2 仅支持云端项目（服务器磁盘），M3 新增本地项目（用户电脑），通过 SSH 隧道访问。本地文件不自动上传到服务器，除非用户主动上传。

**项目类型**：
- **云端项目**：服务器磁盘上的目录，任何客户端可访问
- **本地项目**：用户电脑上的目录，通过 SSH 隧道让云端 Agent 访问

**并行轨道**：
- Track A（服务器）：MCP 桥接网关 + SSH 隧道管理
- Track B（客户端）：Rust 客户端 Core
- Track C（UI）：资源访问可视化

---

## 阶段 3.1 MCP 桥接网关

**目标**：服务器端 MCP 客户端，将 Agent 的资源请求路由到正确的资源节点。

**验收任务**：
- [ ] MCP Python SDK 集成
- [ ] 声明式路由配置（YAML）：
  ```yaml
  routes:
    - prefix: "server://local/*"
      type: local
    - prefix: "client://laptop/*"
      type: ssh_tunnel
      tunnel_id: "tunnel-laptop"
  ```
- [ ] 资源前缀解析和路由分发
- [ ] 安全上下文注入（X-Agent-ID, X-Project-ID）
- [ ] 连接池管理和健康检查
- [ ] 错误处理（连接超时、资源不可用）
- [ ] 单元测试覆盖

**验收实践**：
```python
# 本地资源访问
from neuralswarm.services.mcp_bridge import MCPBridge

bridge = MCPBridge(config)
content = await bridge.read("server://local/home/project/main.py")
# 预期：返回文件内容

# 通过 SSH 隧道访问客户端资源
content = await bridge.read("client://laptop/home/user/project/main.py")
# 预期：通过隧道转发，返回客户端文件内容
```

## 阶段 3.2 SSH 隧道管理

**目标**：管理客户端到服务器的 SSH 反向隧道。

**验收任务**：
- [ ] asyncssh 反向隧道建立（客户端连接服务器，映射本地端口）
- [ ] 隧道注册和发现（客户端注册到路由表）
- [ ] 隧道保活（心跳检测）
- [ ] 断线自动重连（指数退避）
- [ ] 优雅关闭（信号处理、连接清理）
- [ ] 隧道状态查询 API
- [ ] 单元测试覆盖

**验收实践**：
```bash
# 1. 客户端建立隧道
ssh -R 9000:localhost:3000 server@host
# 预期：隧道建立成功

# 2. 服务器查询隧道状态
curl http://localhost:8000/api/tunnels
# 预期：返回活跃隧道列表

# 3. 通过隧道访问客户端资源
curl http://localhost:8000/api/resources/client://laptop/path/to/file
# 预期：返回客户端文件内容
```

## 阶段 3.3 客户端 Core

**目标**：Rust 客户端，连接服务器、执行文件和 Shell 指令。

**验收任务**：
- [ ] Rust 项目初始化（Cargo.toml、项目结构）
- [ ] WebSocket 连接服务器（接收指令、发送结果）
- [ ] 文件读写指令执行（`file_read`, `file_write`）
- [ ] Shell 命令执行（`shell`）— 异步、超时控制
- [ ] 结果序列化和返回
- [ ] 错误处理（文件不存在、权限不足、命令超时）
- [ ] 配置管理（服务器地址、认证信息）
- [ ] 单元测试覆盖

**验收实践**：
```bash
# 1. 编译客户端
cd client && cargo build

# 2. 启动客户端（连接服务器）
cargo run -- --server ws://localhost:8000/ws/client

# 3. 服务器发送文件读取指令
# 预期：客户端读取本地文件，返回内容

# 4. 服务器发送 Shell 指令
# 预期：客户端执行命令，返回输出
```

**通信协议**（服务器 → 客户端）：
```json
{
  "type": "execute",
  "command": "file_read",
  "params": {"path": "/home/user/project/main.py"},
  "request_id": "req-001"
}
```

**响应协议**（客户端 → 服务器）：
```json
{
  "type": "result",
  "request_id": "req-001",
  "status": "success",
  "data": {"content": "..."}
}
```
