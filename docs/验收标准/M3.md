# M3: 资源访问

**状态**：已完成（107 个服务器测试 + 4 个 Rust 测试通过）

**目标**：Agent 可通过 Bridge Router 和 WebSocket 连接访问客户端本地资源（文件、Shell）。支持注册本地项目。

**可见产出**：Agent 能读写客户端本地文件、执行 Shell 命令，桌面端可打开本地项目。

**关键变化**：M2 仅支持云端项目（服务器磁盘），M3 新增本地项目（用户电脑），通过 WebSocket 连接访问。本地文件不自动上传到服务器，除非用户主动上传。

**架构决策**：
- 去掉 SSH 隧道，使用 WebSocket + HTTP
- 客户端主动连接服务器，建立 WebSocket 长连接
- 服务器通过 WebSocket 发送指令，客户端执行并返回结果
- M3 基础 JWT 认证，M6 完整 RBAC
- 两套 API：简单 API（MVP）+ MCP（未来）

**项目类型**：
- **云端项目**：服务器磁盘上的目录，任何客户端可访问
- **本地项目**：用户电脑上的目录，通过 WebSocket 连接让云端 Agent 访问

**客户端 Core**：
- 独立 Rust crate，可编译为独立二进制
- 集成到 Tauri 桌面端
- 通过 FFI 给移动端复用

**并行轨道**：
- Track A（服务器）：Bridge Router + Auth + API
- Track B（客户端）：Rust Client Core
- Track C（UI）：资源访问可视化 + 平台适配

## 阶段 3.1 Bridge Router

**目标**：服务器端资源路由，根据项目类型分发到本地执行或客户端。

**验收任务**：
- [x] Bridge Router 类实现
- [x] 路由逻辑：`server://path` → 本地执行，`client://hostname/path` → WebSocket 转发
- [x] 客户端连接管理（WebSocket 连接池、心跳检测）
- [x] 指令序列化/反序列化（JSON）
- [x] 超时控制（客户端响应超时）
- [x] 错误处理（客户端离线、执行失败）
- [x] 单元测试覆盖

**通信协议**（服务器 → 客户端）：
```json
{
  "type": "execute",
  "command": "file_read",
  "params": {"path": "/home/user/project/main.py"},
  "request_id": "req-001"
}
```

**响应协议**（客户端 → 服务器）：
```json
{
  "type": "result",
  "request_id": "req-001",
  "status": "success",
  "data": {"content": "..."}
}
```

```bash
cd docker && docker compose --profile infra --profile server up -d

# 2. 注册用户并登录

curl -X POST http://localhost:8000/api/auth/login -d '{"username":"test","password":"test"}'
# 预期：返回 JWT token

# 3. 启动客户端（独立二进制）
cd core && cargo run -- --server ws://localhost:8000/ws/client --token <jwt>

# 4. 在 UI 中提交任务："读取客户端的 /etc/hostname 文件"
# 预期：
#   - Agent 调用 file_read 工具
#   - Bridge Router 路由到客户端
#   - 客户端执行并返回结果
#   - UI 显示文件内容
```
